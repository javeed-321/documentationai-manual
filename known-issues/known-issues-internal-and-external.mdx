## Overview

This page lists issues that are known to both internal teams and customers. Each row records the area, the impact, a workaround, and status. Internal-only detail lives on [Known Issues (Internal)](/known-issues/known-issues-internal); the customer-facing view is on [Known Issues (External)](/known-issues/known-issues-external).

<Callout kind="alert">
  A known issue is a confirmed defect with a planned fix. If a behavior is by design, see [Expected Behaviour](/known-issues/expected-behaviour) instead.
</Callout>

## Severity key

| Severity | Meaning | Target |
|----------|---------|--------|
| High | Blocks a core workflow | Fix in the current cycle |
| Medium | Degrades experience | Fix in an upcoming release |
| Low | Minor or cosmetic | Scheduled opportunistically |

<Callout kind="info">
  Fixed issues move to the release notes in [What's New](/whats-new/todo). Check there to confirm whether an issue you hit has already shipped a fix.
</Callout>
